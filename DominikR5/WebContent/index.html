Projekt 1

Jak sprawić, aby rezygnacja z mailingu była odnotowana zarówno we FreshMailu, jak w CRMie?

Należy wejść na konkretną listę odbiorców i wybrać przekierowanie po wypisaniu. Listy Odbiorców -> wybieramy konkretną listę -> komunikatyu i API -> komunikaty. Następnie należy ustawić adres np. na http://mojastrona.pl/wypisany/$$email$$ i wtedy po kliknięciu na link rezygnacji odbiorca zostanie przekierowany na adres, np: http://mojastrona.pl/wypisany/info@freshmail.pl.

Podczas próby połączenia otrzymuję komunikat “Invalid json response, Error code: 0, HTTP code: 0”?

Najczęstszy powód, który generuje taki problem to zablokowanie przez administratora serwera na którym znajduje się Twoja aplikacja wyjścia na port 443.

W jaki sposób pobrać wypisanych odbiorców?

W przypadku każdej listy odbiorców możemy skonfigurować URL na jaki zostanie przekierowany odbiorca po kliknięciu w link rezygnacji. Taki link można personalizować za pomocą adresu email lub dowolnych innych danych które posiadamy w polach dodatkowych.

Jak mogę wyeksportować wszystkie adresy z listy?

Ze względów bezpieczeństwa nie posiadamy takiej funkcjonalności. Można natomiast pobrać dane konkretnych użytkowników znając ich adresy email.

Gdzie mogę znaleźć API KEY i API SECRET?

Ustawienia → integracje i API → zakładka “API”. Zobacz dokładniejszą instrukcję.

Co jest mi potrzebne aby się połączyć z FreshMail’em?

W pierwszej kolejności potrzebna będzie API KEY oraz API SECRET. Dodatkowo należy stworzyć mechanizm do autoryzacji. Jeśli korzystasz z języka PHP to mamy dla Ciebie gotową klasę do autoryzacji na GitHubie wraz z przykładami.

Którą wersję API mam wybrać?

Aktualnie posiadamy 2 wersje API: w technologii SOAP oraz REST. Aktualnie jedyną wersją przez nas rozwijaną jest REST API. Ma ona najwięcej funkcjonalności, jest najprostsza w implementacji oraz jest najwydajniejsza. Zalecamy korzystanie właśnie z tej wersji API.
